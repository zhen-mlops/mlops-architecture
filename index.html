i 📁 Your ML Repo
Code repository
A GitHub repository holding all your ML project code — training scripts, data processing, configs, and CI/CD workflow definitions. Every change starts with a commit here.
i .github/workflows/
CI/CD pipeline definitions
YAML files that tell GitHub Actions what to do when you push code. Example: "on push to main, trigger my Kubeflow pipeline via the self-hosted runner."
i notebooks/
Jupyter Notebooks
Interactive .ipynb files for experimentation — data exploration, model prototyping, visualization. Can be executed automatically by GitHub Actions.
i src/
Python source code
Production-ready Python modules — reusable functions for data loading, training, and evaluation. Clean importable code designed for automated pipelines.
i ⚡ GitHub Actions
CI/CD Pipeline Executor
GitHub's built-in automation engine. Reads your workflow YAML on push and runs the steps — connects to your self-hosted runner to trigger Kubeflow pipelines on your local machine.

🖥 Computer 1 · Your Physical Machine / VM @ CSC
Hardware click
Your physical CPU, RAM, disk. Everything runs on top of this — your own laptop/PC or a CSC virtual machine.
Low-level Virtualization (if VM) click
If using a CSC VM, this layer emulates a full computer in software (KVM/QEMU), giving you a dedicated machine on shared hardware.
Operating System click
Ubuntu Linux (or WSL on Windows). The base OS managing hardware and running Docker + all software on top.
Docker — Containerization click
Packages each app into an isolated container with all dependencies. Lighter than a VM but provides similar isolation for Kubeflow, MLflow, etc.
KiND — Kubernetes in Docker click
Runs a full Kubernetes cluster inside Docker on your machine. Creates K8s "nodes" as containers — gives you a real K8s environment locally for all MLOps components.

MLflow
Model Tracking & Registry
Experiments · Versions · Artifacts
Records every experiment — hyperparameters, metrics, and trained model files. After Kubeflow's pipeline trains a model, it stores the result in MLflow. KServe later pulls models from here to deploy. UI at localhost:5000.

KServe
Model Serving
Inference API · Auto-scaling
Takes a trained model from MLflow and exposes it as a REST API. Send data via HTTP, get predictions back. Runs on Kubernetes with auto-scaling — spins up more copies when traffic increases.
